Introduction

Backpack.js is a lightweight Backbone extension that provides easy-to-use, extensible UI components, and was built by a couple of Airbnb Engineers.

Backpack.js has been built with a focus on the creation of components - CSS files have been included to help you get up and running, but feel free to modify styles as necessary.

After building the Airbnb mobile web app with Backbone, we thought it would be nice to have a lightweight pack of components that we could drop in to any Backbone project.

This initial release is the result of our work abstracting components that we initially imagined and built for the Airbnb mobile site. We would like to open this initial release to the community to hear what you guys think and welcome any contributions.

Overview

Backpack.Component

Component is the base View which all Backpack.js components extend. It provides convenient helper methods for manipulating the component and itâ€™s content, many of which follow a syntax similar to jQuery in that methods can be chained, or alternatively can be passed in upon initialization of the component.

Component provides the following convenience methods to all components:

  • render
  • show
  • hide
  • close
  • remove
  • before
  • after
  • append
  • prepend
  • content
  • setContent
  • parent
  • addClass
  • removeClass
  • name
  • slug

Backpack.Tabs

Tabs are a grouping of TabItems, responsible for automatically displaying the content associated with any given tab. Clicking on a TabItem automatically displays the content, and can also be extended to execute any number of given arbitrary functions. 

var tabs = new Backpack.Tabs({
                show: true,
                add: 
                  [
                    {
                      content: 'Tab 1',
                      tabContent: 'Some tab 1 content'
                    }
                  ]
              });
var tabs = new Backpack.Tabs({
                show: true
              });

    tabs.add({
            content: 'Tab 1'
            tabContent: 'Some tab 1 content'
    });

Backpack.Menu

Menu is a means of displaying many MenuItem objects. A MenuItem can consist of text, or any amount of arbitrary HTML (images, for example). When clicked, a MenuItem is capable of redirecting a user to a simple URL or executing any arbitrary function. 

var menu = new Backpack.Menu({
      // the parent is the modal the
      // example menu launches in
      parent: this, 
      show: true,
      add: 
        [
          {
            content: 'Menu Item 1',
            // events can be something like this on click
            events: '#Menu'
          },
          {
            content: 'Airbnb',
            // events can also be a normal link on click
            events: 'http://airbnb.com',
            // bump adds a 'bump-right' class which
            // floats the element right
            bump: 'right'
          },
          {
            content: 'Menu Item 2',
            // events can also be a function on click
            events: function() {
                      return console.log('test');
                    }           
          },
          {
            content: 'Menu Item 3',
            // or events can be a normal
            // events hash
            events: {
              'mouseenter': function() {
                console.log('mouseenter');},
              'click': function() {
                console.log('click');}
            }                       
          },
        ]
  });

Backpack.Modal

Depending on the properties a Modal object is initialized with, they can be easily configured to provide two different experiences - Popups are notifications presented in the center of a userâ€™s screen, and Lightboxes are notifications presented in a similar manner, with the exception that the background content becomes dim, drawing attention to the middle Modal. 

var popup = new Backpack.Modal({ 
                  content: "This is a simple popup modal.",
                  show: true
            });
var lb = new Backpack.Modal({
                content: 'This Modal is a bit fancier.',
                show: true,
                // set the overlay color
                color: 'rgba(255,255,255,0.6)',
                lockOverlay: true, 
                // put a 'x' in the corner to close
                closable: true
         });

Getting Started

git clone git@github.com:airbnb/backpack.js.git
cd backpack.js && npm install


Cake Build Tools

cake compile        # Compile CoffeeScript source files

cake build          # Creates /lib_path/Backpack-bundle.js & 
                    # /lib_path/Backpack-bundle.min.js & 
                    # /lib_path/js/*.js

cake test           # Opens Jasmine SpecRunner. Watches
                    # BackpackSpec-Bundle.js for changes

cake docs           # Generate annotated source code 
                    # with Docco

cake watch          # Recompile CoffeeScript source files when 
                    # modified to Backpack-bundle.js

cake watch:js       # Recompile CoffeeScript source files when 
                    # modified to individual .js files


Testing

Backpack.js tests are written using jasmine with sinon.js and jasmine-sinon.

You can run the test suite with 
cake test
